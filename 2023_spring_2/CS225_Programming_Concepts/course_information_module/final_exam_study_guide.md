# Final Exam Study Guide

Study Guide for CS225 Final Exam – Online Sessions

 When:

2 hours, between Monday and Sunday of Unit 8.

It may be a good idea to take the exam by Saturday (or even Friday) to give you some buffer time, just in case something may happen in the last minute. 

It’s also a good idea to set aside a 2.5-hour window for the actual test, as the pre-test check-in may take at least a few minutes. 

Format: 

Allowed: handwritten notes on either a 5x7” card or a half sheet of 8.5 x 11” paper (US letter size: 8.5 x 11''). You may use both sides. You must show your notes to the camera, two seconds per side at the beginning of the test as part of the room scan.

No other resources are allowed. You will only be allowed to open the final exam (a longer quiz) and type up your answer on the quiz page. You will not be allowed to open additional browser tab/window or use additional software/devices.

No scratch paper. You may scratch in the top (or bottom) portion of the Editor window for a question, as long as you mark it clearly. One example:

---- my scratch. ignore ----
 xxxx
 xxxxxx
---- end scratch
---- Work starts here. ----
my work to be graded xxxx 
my work to be graded xxxx

You're strongly recommended to use "Preformatted" format for your code work, as it displays indentation clearly and really helps you trace your work. 

Online Proctoring Requirement

Be in a private, well-lit room free from other people and noises;

Clear your workspace. Please show your workspace during the room scan of your exam session;

No break is allowed during the test. Refresh yourself before starting the exam.  Please follow  [link: https://www.park.edu/terms-and-regulations/disability-guidelines/] Park's Disability Guideline if you require accommodation.

A practice quiz has been set up to help you get acclimated to the online proctoring system. Please find it through the Honorlock link in the left-hand side Canvas menu. The practice quiz is optional and may be taken repeatedly.

Final Exam “Show one question at a time” Setup

The final exam is set up to display one question at a time. Unit 6 and Unit 7 quizzes are set up in this style to help you prepare for it. 

You may use the Next/Previous buttons at the bottom of the page or the sidebar on the top right corner of the page to navigate through questions. Questions are not locked so you may go back to an answered question and revise as often as you need. See the Canvas Student Help page  [link: https://community.canvaslms.com/docs/DOC-10535-421291043] “How do I take a quiz where I can only view one question at a time?”

Types of questions:

10 - 15 multiple choice questions: one single correct answer

 two to four short answer questions: explain concepts, show the result of code, or write short segments of code, and

 two to three coding questions: write a complete method/class/program

Topics: 

Comprehensive (i.e. units 1 – 7).  Review all quizzes, assignments, and discussion questions.

Here are the overall learning outcomes for the course, which provide a big-picture guide for final exam questions (hint: especially the coding questions and short-answer questions):

Given a problem description, implement a C++ program that uses inheritance and operator overloading.

Implement functions with call-by-value, call-by-reference, and default parameters.

Use pointers, references, and both 1- and 2-dimensional arrays.

Compare C-style arrays of characters with C++ strings.

Analyze a C++ program to find syntax and logic errors.

Additionally, there may be questions on:

Fig 1-2 Processing of a C++ program and relevant paragraphs (Unit 1/ch1)

trace the results of a program involving the vector class and its iterator (Unit 6/ch21). You're expected to know vector's default constructor, copy constructor, size(), at(), [], begin(), end(), and ++/-- of vector iterator. A brief reference of additional member functions (like the ones listed in lecture PPTs) will be provided should they be needed in a question.

when the C++ string class is needed, you're expected to know size()/length(), at(), and []. A brief reference of additional member functions will be provided should they be needed in a question.

There won't be questions on:

output format manipulators (Unit 1/ch3)

working with input/output stream objects (Unit 1/ch3) except for operator overloading and using initialized stream objects with << and >>

knowing which library header file to include

writing a function template or class template (Unit 2/ch13, Unit 6/ch13)

Coding Requirements

Use proper syntax and style in your coding.

Comments are not required. On the other hand, they may help the instructor understand your code especially if your solution uses an “innovative” algorithm.

Make sure that the output from your code produces the exact output shown in the problem description.

Change the font style of your code to Preformatted style (as explained in Unit 1 Discussion). This may be helpful when checking your work etc. 

How to prepare for the final exam: 

Start those from the first day of this course:

Practice coding on paper or in a plain text editor,

Practice reading and tracing code on paper or screen w/o using any compilers,

Practice coding with minimal reference to course material and the text, especially after you’ve completed a relevant unit.

While no IT professionals are expected to program without using references/Internet, they are expected to know a language and programming fundamentals to a decent level. Those come not through memorization, but after many practices and problem-solving sessions. That’s why on-site job interviews for software-related positions often expect applicants to code on a whiteboard without a computer/Internet/phone a friend.
