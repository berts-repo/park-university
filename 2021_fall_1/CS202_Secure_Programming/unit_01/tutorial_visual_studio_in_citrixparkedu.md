Accessing your Local Drive from the Citrix Environment

To see your local drive in the Citrix environment you may click the Windows icon in the bottom left corner of the Student Lab Desktop:

This will open the Start Panel:

By clicking the “This PC” Icon or the “File Explorer” Icon you will obtain a File Explorer Windows that will show both, the folders in the Citrix environment, as well as all the local file sources attached to your computer, including hard drives, USB drives, and other detachable media. These local devices will be listed under “This PC” on the left side of the panel, and also under “Network Locations”.

Their labels may appear a little odd, but the first letter is the drive letter that it has been assigned on your local computer. For example, in the figure below, drive C$ (\\Client) (V:), is the hard drive of your local machine C:. Do not confuse it with Local Disk (C:), that is the Virtual hard drive in the Citrix Environment. In a similar manner, drives D$ (\\Client) (U:), and E$ (\\Client) (E:) in the figure below, correspond to the local drives D and E, possibly USB drives in the user’s computer.

You may use this Window like any other File Explorer to navigate, copy, and paste folders and files between the Citrix Environment and your local drives.

A Final Note regarding the use of Citrix Park Virtual Lab

Citrix Park Virtual Lab is a distributed system working in the Cloud. At any given time, there could be more than one server machine providing access to Citrix. Depending on various factors, like where the user’s machine is, the load of the Internet, or the availability of servers, among others, users may get access to different copies of the Citrix environment. All these copies have the same software and capabilities, but if the user saves files in one of them, they will not be present in the other copies. This may produce strange situations, if the user expects to see these files again. A user may create a file in a first session, but it may not show up in the following sessions, eventually re-appearing when s/he least expects it. The explanation is that the file was created in the first session with a Citrix copy that was not used in subsequent sessions, just to reappear at a later stage. Be mindful that we are talking here of files stored in the Virtual environment. Files stored in your local hard drives or USB drives should remain there, unless you delete them. Unfortunately, there is no way to request a specific copy of the Citrix environment at login, and even if a user uses the same computer in the same place from session to session, there is no guarantee that the same copy will load. 

Some other times, files created in a session may be permanently destroyed after the end of that session. This may happen after maintenance work done on the Citrix environment.

The same situation explains why a user may have to use Microsoft Visual Studio as if it were the first time more than once. If the user gets a copy of Citrix different than the one where s/he initialized Microsoft Visual Studio before, s/he may have to do it again. And of course, s/he will also have to re-create any previous Microsoft Visual Studio project s/he may have had saved in the Citrix environment.

The moral of the story is this: Always get backups of your work in a local drive (USB or Hard Drive in your own machine) before a Citrix session ends. You may need this copy in the following Citrix session. Do not expect to find it in the Citrix environment. You may or may not.
