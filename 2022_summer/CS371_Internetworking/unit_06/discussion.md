In section 14.5.5 it talks abouts about load balancing between next hop routing. The idea is if there are two different next hop routers with the same cost there will be a mechanism that that forwards the packets between both of the routers to the destination prefix. OSPF automatically load balances if the cost is the same, while only EIGRP supports unequal cost load balancing. Static routes can be configured to load balance if both routes have the same destination prefix and the same configured AD. CISCO mechanism is CEF; "Cisco Express Forwarding load balancing is by default using source and destination IP to calculate the hash and distribute traffic over multiple paths." (Popeskic) This can be configured with per-packet or per-destination load balancing. 

 

Popeskic, Valter. (2016, January 29). Static Route Load Balance. How Does Internet Work. Retrieved July 17, 2022, from https://howdoesinternetwork.com/2015/static-route-load-balance#:~:text=If%20you%20have%20two%20routers,but%20with%20different%20next%2Dhops.
