I want to talk about FHRP, but specifically load sharing with HSRP. As we learned, HSRP creates router redundancy by creating a virtual router group to an outside network, in the case the router fails. Our online textbook briefly talked about load balancing between devices, so I decided to explore this with some outside sources. According to CISCO documentation I found some interesting side notes on load balancing; depending on how you configure the router, packets that return from the servers may or may not use both return paths. Also, packets that return from the servers to the hosts do not need to go through the active router. I also learned that some ethernet controllers, including QUICC and Lance, can only have one MAC address. These platforms only permit one HSRP virtual router group, and load balancing on these platforms is not possible with HSRP.
