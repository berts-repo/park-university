# Unit 3: From E-R Model to Design

Introduction

This unit will teach students how to create a relational data model based on a pre-existing Entity-Relationship Diagram. The relational data model is the actual design of the database in paper, ready to be implemented in a relational database. Students will learn the correct way to transform entities, attributes and relationships into relational tables. They will also be introduced to the idea of database normalization, a technique that minimize redundancy in databases, and makes them more efficient.

Unit Learning Outcomes

At the conclusion of the unit, the learner will be able to:

Design a Relational database from and Entity Relationship (ER) Diagram (CLO3).

Distinguish between various kinds of tables, fields and records in a relational database design (CLO 1, CLO3).

Distinguish between primary integrity and referential integrity (CLO3).

Distinguish between first, second and third normal forms for a relational database (CLO 3).
