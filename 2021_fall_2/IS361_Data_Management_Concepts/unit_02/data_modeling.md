# Unit 2: Data Modeling

Introduction

In this unit, we will learn how to create Entity-Relationship Diagrams (ERDs). These diagrams represent the entities, their attributes, and their relationships found in real-life systems. ERDs are the starting point in the database design process. They are prepared during the initial analysis stage of the design, and are refined and perfected as the users of the system are involved in their creation. Once completed and approved by users and designers they will be used to construct the database model for the system. This unit will teach students how to read, understand, and create these diagrams.

Unit Learning Outcomes

At the conclusion of the unit, the learner will be able to:

Create an Entity Relationship Diagram (ERD) from a set of written requirements (CLO2).

Distinguish between various kinds of entities, attributes and relationships in a database design (CLO 1, CLO2).
