I will be responding to prompt 3: Explain with a concrete example (not from the textbook) how Views can be used to protect data.

A view gives a user access to a portion of the base table. It is a predefined query that a user can access for a subset of data.  It is used when there is sensitive information that you don't want a user accessing inside a database. It gives the user access to the data they need but not the entire table or tables.

I would imagine a hospital would use views for their employees. A front desk receptionist might only need the name, address, and phone number of a patient. Where the billing department would need access to their insurance and possibly income. So, a view would be created to query just give the patient name, address and phone number for the receptionist, compared to their income based information.
