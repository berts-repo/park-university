Late Assignment Policy

Overall

You’re expected to complete course work on time. This is especially important for programming courses where topics are built on top of materials covered previously. Our 8-week term is already tight enough. As a context, a 3 credit hour course means 3 hours in class plus 6-9 hours of study time each week in a standard 16-week semester. You need to double that time devotion for an 8-week course.

zyBook PA and CA

zyBook will automatically record the percentage of the activities you have completed. While you can continue working on those activities past due date, you will receive credit only for the work you have completed by 11:59 pm Central Time on each Sunday.

Discussions

No late posts will be accepted past Sunday due dates as no one else will be discussing then.

Assignments

Assignments should be submitted by their due dates. Instructors may grant exceptions on valid documented excuses (see Course Policies). Check with your instructor for specific late policy on assignments. You should always notify the instructor of such circumstances as soon as possible. With or without valid reasons, notify the instructor of your late submission otherwise it may not be graded in time.     

Quizzes

No late submissions are allowed on quizzes, as the answers will have been posted after the due dates.

Introduction

This unit will cover the basics to build a Java program using variables, basic input/output, control structures and procedural design. Different from Python, Java requires a variable to be declared as a specific type before it can be used. The concepts of control structures (selection, loops) and procedural design (user-defined methods) are the same as in Python and most programming languages therefore this unit may be like a review most of the time. Nevertheless, pay attention to syntax differences and practice coding if this is your first Java course.

 Introduction

Software commonly needs to manage a list of items, such as a course roster or pictures on your smartphone. Arrays and ArrayList can be used for such purpose. In Java, an array is a special object that can hold a list of elements of one same type. We will learn some common array manipulation techniques. Array is a built-in language feature of Java (and many other programming languages). ArrayList is a Java API class and provides methods to store, retrieve, and manipulate elements in an ArrayList object. While ArrayLists can only store reference type elements (i.e. objects), arrays can store primitive values or objects.

  Introduction

Searching and sorting are common operations on lists of items. This unit will cover liear search, binary search, selection sort and insertion sort. To help us compare the performance of different algorithms, we will introduce Big O notation, a mathematical way of describing how a function (running time of an algorithm) generally behaves in relation to the input size.

The arrays we introduced in the last unit are referred to as one dimensional arrays. Arrays of higher dimensions are used in software too. For example, two-dimensional arrays are commonly used to implement matrix and 2D images. We will learn the basics of two-dimensional arrays and their processing in this unit. The same principles apply to higher-dimensional arrays.

A programmer-defined class allows us to bundle a grouping of data (which becomes data members of a class) and operations that can be performed on that data (which becomes method members of a class). This is called encapsulation, the #1 principle of OOP. This unit will cover how to define a class, including declaring data members, method members, and constructors.

Introduction

This unit introduces another two of the main principles of object-oriented design: inheritance and polymorphism (the #1 principle of OOP is encapsulation, which we covered in the last unit). Inheritance allows us to create new classes from existing classes. A new class built via inheritance (often referred to as subclass, child class, or derived class) is a more specific version of the existing class (known as super class, parent class, or base class). Polymorphism refers to the ability for the same code to be used with different types of objects and behave differently.

ULO #2: Distinguish overload and override. (CLO 3)
