# Study Guide for CS252 Final Exam – Online Sessions

When

2 hours, between Monday and Sunday of Unit 8. May be taken only once.

It may be a good idea to take the exam by Saturday (or even Friday) to give you some buffer time, just in case something may happen in the last minute. 

It’s also a good idea to set aside a 2.5-hour window for the actual test, as the sign in may take at least a few minutes. 

 Format

Allowed: handwritten notes on a 5x7'' card (or a half sheet of letter-size paper <letter size: 8.5 x 11''>). You may use both sides. You must show your notes to the camera, two seconds per side at the beginning of the test as part of the room scan.

No other resources are allowed. You will only be allowed to open the final exam (a longer quiz) and type up your answer on the quiz page. You will not be allowed to open additional browser tab/window or use additional software/devices.

No scratch paper. You may scratch in the top (or bottom) portion of the Editor window for a question, as long as you mark it clearly. One example:

---- my scratch. ignore ----
 xxxx
 xxxxxx
---- end scratch
---- Work starts here. ----
my work to be graded xxxx 
my work to be graded xxxx

You're strongly recommended to use "Preformatted" format for your code work, as it displays indentation clearly and helps you trace your work.

Online Proctoring: 

Be in a private, well-lit room free from other people and noises;

Clear your workspace. Please show your workspace during the room scan of your exam session;

No break is allowed during the test. Refresh yourself before starting the exam.  Please follow  [link: https://www.park.edu/terms-and-regulations/disability-guidelines/] Park's Disability Guideline if you require accommodation.

A practice quiz may be available for you to get acclimated to the online proctoring system. Find it under Honorlock in the Canvas menu, otherwise check with your instructor. The practice quiz is optional and may be taken repeatedly. Be aware that the practice quiz is just a standard practice quiz from the proctoring system. Its quiz settings and question types don't match those of the final exam. 

Final Exam “Show one question at a time” Setup

The final exam is set up to display one question at a time.  Unit 6 and Unit 7 quizzes are set up in this style to help you prepare for it. 

You may use the Next/Previous buttons at the bottom of the page or the sidebar on the top right corner of the page to navigate through questions.  Questions are not locked so you may go back to an answered question and revise as often as you need. See Canvas Student Help page  [link: https://community.canvaslms.com/t5/Student-Guide/How-do-I-take-a-quiz-where-I-can-only-view-one-question-at-a/ta-p/482] “How do I take a quiz where I can only view one question at a time?”.

Types of questions:

10 - 15 multiple choice questions: one single correct answer

 two to four short answer questions: explain concepts, show the result of code, or write a short segment of code, and

 two to three coding questions: write a complete method/class/program

Topics

Comprehensive (i.e. units 1 – 7).  Review all quizzes, assignments, and discussion questions.

Here are the overall learning outcomes for the course, which provide a big picture guide for final exam questions (hint: especially the coding questions and short-answer questions):

Given a problem description, implement a Java program that uses procedural programming constructs.

Given a problem description, implement a Java program that uses object-oriented programming constructs.

Given a problem description, implement a Java program that uses inheritance or polymorphism.

Use arrays (1 and 2 dimensional) and ArrayLists to store and manipulate collections of data.

Compare methods to solve searching and sorting problems.

GUI programming and exception handling are important topics based on OOP and inheritance. Be prepared for questions on those topics in the final exam.

GUI: use common GUI components (the ones used in a windowed application with buttons, labels, and textfield type of controls); event handling

Exception handling: how exception handling works in general (try-catch-finally, throws clause <in method header>), identify & handle common exceptions

 Coding Requirements

Use proper syntax and programming style in your coding.  

Comments are not required.  On the other hand, they may help the instructor understand your code especially if your solution uses an “innovative” algorithm.

Make sure that the output from your code produces the exact output shown in the problem description.

Using the Preformatted style for code work (as explained in Unit 1 Discussion) is preferred. It makes your code easier to read for yourself and for grading.

How to Prepare for the Final Exam

Start those practices from day one of this course:

Practice coding on paper or a plain text editor,

Practice reading and tracing code on paper or screen w/o using any compiler,

Practice coding with minimal reference to course material and the text, especially after you’ve completed a relevant unit.

While no IT professionals are expected to program without using references or the Internet, they are expected to know a language and programming fundamentals to a decent level. Those come not through memorization, but after many practices and problem-solving sessions. That’s why on-site job interviews on software-related positions often expect applicants to code on a whiteboard without a computer/Internet/phone a friend.
