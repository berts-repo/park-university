# Course Overview

Course Description

This course presents the theory of operating systems and an overview of one or more operating system environments. Operating system concepts covered should include (but are not limited to): process management, memory management, I/O management, file management, and security.  Theory concepts will be put into practice with exercises, some requiring college algebra skills and/or basic programming knowledge. Operating system environments may include (but are not limited to): Windows, UNIX, and Linux.

Prerequisites: CS319 and  either CS202 or CS225
Credit Hours: 3:0:3

Core Learning Outcomes

General concepts: Explain the basic concepts and functions common to all operating systems. Explain the basic concepts and functions common to all Operating systems. Examine the concepts of security, including cryptography, user authentication, attacks from inside and outside the system, and protection mechanisms.

Process Management: Examine the concepts and techniques of process and thread management, including interrupts and multitasking. Discuss CPU scheduling algorithms. Examine the concepts of deadlock, including deadlock detection, deadlock recovery, deadlock avoidance, and deadlock prevention.

Memory Management: Examine the concepts and techniques of memory management such as virtual memory, paging, segmentation, and page replacement algorithms.

File and I/O Managements: Examine the concepts and techniques of file systems, including files and directories. Examine the concepts and techniques of Input/Output, including I/O hardware and software.

Overview Of The Course Learning Activities

In each unit you will have regular learning activities:

Readings: Textbook and supplemental materials
Lectures: Powerpoint Presentations
Individual Homework Assignments: There will be end of chapter questions and problems assigned in each unit for the student to complete.
Discussions: Post in each graded topic at least for that unit, and post a response to someone else's 

On certain units you will have additional learning activities:

Quizzes: In some units will have multiple choice quizzes.
Linux Lab Assignments: Each student will receive an remote account on the Linux server located at the main Park campus, and there will be assignments requiring students to write computer programs, then compile and run them on the server.
Exams: Complete the final exam. 
Final Exam: Short essay questions that are very similar to the unit assignment questions and problems.

The final exam is closed book but you may have a single page of notes and a calculator:

one page note sheet: letter size (8.5 x 11”), hand-written, double-sided

the built-in calculator on your computer, but only the standard/basic mode is allowed. No scientific/programmer/advanced modes. Mac users should only use the calculator app at /Applications/Calculator.

Use of the Internet is not allowed during the exam

Assessment Of Course Objectives

For this course, the assessment artifact is a final exam which counts for 20% of the grade. In the artifact there will be 4 questions in each of the categories.  (Critical Thinking: Synthesis, Analysis, Evaluation; Content: Terminology, Concepts, Application; and Technical Skills: Whole Artifact, Component, Relationship) Thus, there will be 35-40 questions total in the final exam artifact. The artifact will test all four of the core learning objectives in each category.
