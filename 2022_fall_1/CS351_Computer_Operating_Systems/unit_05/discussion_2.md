Class,

This simulation represents the time it takes to access a block of data that is stored on secondary memory. The graph is read from top to bottom, where the y axis is the percent of the process completed. The dotted lines represent blocks 1 through 10, and from left to right. We direct the simulation to have a head (start) position at 100, and it then the simulations makes a line to the next block according to the selected algorithm. In the First Come First Serve algorithm the blocks are read in sequential order, and it does not matter where the next block is stored, it just goes to the next block in the read sequence. In the Shortest Seek Time First algorithm the block order does not matter, it just finds the closest block in the read sequence.

The algorithm with the least head movement in the simulation goes to Shortest Seek Time First. I came to this conclusion because it holds the least cross over time, in which, it does not often have to cross back over a block that it has read to read the next block in the sequence. You can see it in the simulation with the zig zag pattern. The perfect simulation would be where there is no cross over.
