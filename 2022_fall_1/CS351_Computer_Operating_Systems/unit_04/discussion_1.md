In 1983 Microsoft announced their GUI, Windows, that was now to be packaged with their operating system MS-DOS, which was currently being shipped with IBM PC compatible systems. This new product line split into two OS families, each with their own file system: the windows 3x and 4x family, and also Windows NT. (History of Microsoft Windows)

The 3.x and 4.x family includes Windows 3.0, Windows 3.1x, Windows 95, Windows 98, and Windows Me. Windows 3.0/3.1x and Windows 95 all used FAT16B file system, where windows 98 and Me used the FAT32 file system. (List of default file systems)

Modern OS are based on the Windows NT family. (History of Microsoft Windows) Windows NT 3.5/4.0 were the first systems to implement the modern NTFS file systems and is also used in many other Windows OS variations, including Windows Server 2003/2008, Windows 2000/XP/Vista/7/8/10/11. (List of Default File Systems)

Linux systems are compatible with Windows file systems because of the mounting file system but not all the metadata is compatible between the systems, for e.g. the permissions and file names. There are Linux distributions that were designed to be fully implemented with the FAT file system come from the UMSDOS family, e.g. MuLinux, Monkey Linux, and WINLinux. (Fat Filesystem and Linux) Modern Linux system can also mount NFTS and use them for organization.
