what makes a good file system is the ability to organize files/directories and store metadata about them. By having the files centralized, it can increase security and productivity (e.g. permissions and organization). A file system in modern multiprogram OS needs to be able to also keep information about multiple users, the configurations, while keeping redundancy to a minimum. Each user should be able to read, write, create, delete, and modify a file, but the file system also might need to share this files as well. 

A file management system might not always be the best implementation. While doing online research for this discussion, I found an articles that talks about the benefits of using a DBMS over a file management system for databases.  A database management system (DBMS) has no redundant data through a process called normalization, and it keeps the data flexible, easy to recover, easy to access, private and secure compared to a file management system. (Singh 2022)

 

Singh, C. (2022, July 24). Advantages and disadvantages of DBMS: DBMS vs file system. BeginnersBook. Retrieved September 8, 2022, from https://beginnersbook.com/2015/04/dbms-vs-file-system/
