Class,

I wanted to talk about RDP vulnerabilities with the Windows Remote Desktop, and how there has been a steady increase in cyber crime in regards to this protocol. In 2018 ransomware was leveraged by CrySIS and SamSam to exploit this RDP protocol. The fallowing year three more exploits were discovered, one being "wormable" called BlueKeep. (Nahorney 2020) This many vulnerabilities has caused a spike in attention in regards to attack vectors and the defense of networks that uses this protocol: Windows Remote Desktop uses TCP port 3389 to gain remote access to a node. This is where all the attention is, as 80-88% of networks will get get a positive signature for someone preforming reconnaissance on that port, and 30-31% will see the same signatures monthly. (Nahorney 2020) The solution to the port 3389 vulnerabilities is to just not use Windows Remote Desktop, but if you have to can change the ports. (Nahroney 2020) Another trend to this solution is have your framework in the cloud with SaaS services, as this is the most affect way to keep a system up-to-date. (Security Outcomes Study 2020)
