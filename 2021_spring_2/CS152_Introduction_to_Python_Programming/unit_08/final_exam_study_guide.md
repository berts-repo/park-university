# Final Exam Study Guide

Exam Notes

This exam is closed book and closed internet access, except for exam itself. However, you may bring one, handwritten 8.5x11” sheet of notes (both sides). For scratch paper, you are allowed a small whiteboard, or one blank piece of scratch paper inserted into a plastic sleeve, and a dry erase marker. You will be required to show your scratch paper to the proctor and erase it on camera before you will be permitted to exit the testing session.

There 20-25 questions, worth 350 points, including multiple choice, True/False, short answer, and coding.

You have 2 hours to complete the exam. You may only take the exam once.  

Do not spend a lot of time on a 5-point question and then leave a 50-point coding question blank because you ran out of time.

During the exam, you will be expected to type in short programs/functions directly in Canvas. You will not be allowed to use a compiler, nor will you be allowed to type in another editor and copy/paste into Canvas. The code you type does not have to compile and run for full credit. Minor syntax mistakes are irrelevant. When typing your answers, set the style to ‘Preformatted’ and use spaces to indent.

Exam Content

The exam covers Chapter 1-12 from the text. The core learning outcomes for the course are

Implement a Python program that uses iteration, branching and function calling based on a written problem description.

Analyze a Python program to determine syntax, run time, and logic errors.

Implement Python functions with parameters.

Use collections such as lists, strings, tuples and dictionaries in a Python program.

Access text files for input and output.

So you can expect to see questions related to these topics on the exam. You will be asked to read code, describing what it does and finding errors. You will also be asked to write short pieces of code, sometimes completing code that is already there and sometimes from just a description. While turtle graphics is not a core learning outcome, it was used to illustrate loops, branching and function calls, so you can expect there to be some questions referring to turtle methods as well.

As a good review, you should look over all of your assignments and make sure you understand what you have written. Also look at the practice problems you have done in the text and at various practice sites, like codingbat.com or codestepbystep.com.
