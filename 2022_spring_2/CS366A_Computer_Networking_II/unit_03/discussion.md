I discovered and wanted to explore extended-pings and extended-traceroute. As far as I know, these tools can only be preformed on a Cisco device, as you have to run the commands in privileged EXEC mode from the router's CLI. Extended ping allows the source IP addressed to be changed to any IP address on the router. Extended traceroute commands are used to see the route that packets take to get to their destination. This is helpful when troubleshooting loops, or determining where packets are getting lost. There are more options for both of these commands, and they can be found in the link below. 

 

https://www.cisco.com/c/en/us/support/docs/ip/routing-information-protocol-rip/13730-ext-ping-trace.html?referring_site=bodynav
