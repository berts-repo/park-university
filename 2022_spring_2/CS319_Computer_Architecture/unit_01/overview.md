# Unit 1: Overview

Introduction

Our study will start from the concept of computer architecture. We will explain the main components and functions of computers. 

We will then study the machine-level representation of data, i.e. how data is represented inside of computers using 0s and 1s. We will explain how numeric data is represented in computers in general and introduce Two’s Complement representation, the main representation for signed integers. We will learn how arithmetic is carried out on signed integers in Two’s Complement representation. That is the basis of the Arithmetic and Logic Unit (ALU), an essential circuit in CPUs.

Extra: in case anyone wonders why we need to know how data is represented in 0s and 1s, the  [link: https://www.acm.org/binaries/content/assets/education/cs2013_web_final.pdf] Computer Science Curricula 2013 by ACM and IEEE Computer Society recommends machine-level representation of data as a required topic for the Architecture
and Organization knowledge area, which "develops a deeper understanding of the hardware environment upon which
all computing is based, and the interface it provides to higher software layers." Read the section on what Architecture and Organization (AR) should cover (page 65-66, the page number displayed in the top navigation bar) and a breakdown of the topics in "AR/Machine Level Representation of Data" on (page 67).

At the conclusion of the unit, the learner will be able to:

Describe the main functions and structural components of a computer. (CLO 1)

Convert unsigned integers and fractions between decimal, binary, and hexadecimal number systems. (CLO 1)

Convert signed decimal integers to/from two's complement representation. (CLO 1)

Perform two's complement arithmetic (negation, addition, and subtraction) (CLO 1)
