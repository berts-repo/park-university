In our text it discusses passwords and the importance of password strength.  In some organizations we have seen password length grown from a minimum of 8 to 12 or 16 characters and require the inclusion of numbers and special characters.  One chief complaint is that when passwords must be so difficult employees will typically write them down thus defeating the purpose of something you know giving way to the use of passphrases and virtual password.  After reviewing the textbook passage on Something a Person Knows" offer your opinion on how you think passwords should be required.  Should they be of a minimum length, numbers, special characters, how often do we change them, and any other nuance you would like to share.

Passwords need to be easy-to-use and manage, and the average user uses the same password or variation for everything. It is also unfortunately common for a database leak, and it could contain personal emails and passwords. If a user is using the same password and email as the database leak then there is now a larger attack surface for the threat actors to exploit. This is why it would be best practice to use a two-factor-authentication. 

I personally switched to a password manager over a year ago, and it has made everything so much more convenient. It is cross-platform compatible, and I only have to remember and manage one password. I can then use the password manager to generate passwords which I don't have to remember. I think these managers should become more mainstream as it will greatly increase password strength.
