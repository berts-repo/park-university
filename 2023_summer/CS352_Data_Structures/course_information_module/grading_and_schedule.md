Introduction

This unit is an introduction to algorithm analysis – how you compare two ways of doing something. As a non-computer example, if you need to move the contents of your apartment across town, you could put one item in your car, drive over, drop it off, then drive back, and repeat. This however is pretty time consuming. A more time-efficient approach is to rent a moving van, put everything in the van, make one trip, and then unload everything. But this has a higher cost, perhaps, because you have to rent the van. A third approach is to fill your car with as much as possible, so you still make several trips, but not as many as in the first case. What makes this more interesting is when you consider different versions of the problem, like moving across country or just down the block. There often is no best universal best answer, but the analysis is important to choose based on what criteria are most important in each individual case.

The discussion challenges you read about and summarize different combinatorial algorithms. Some of these may be familiar but others won’t, but by the end you’ll have seen many ways of accomplishing many things you may not have even considered before. The project is the first of two parts where you build the infrastructure to explore a two-dimensional space efficiently. This unit you build a very simple version of the program, and then in a subsequent unit you expand the search algorithm to handle a more general case of the problem. 

Unit Learning Outcomes

ULO #1: Implement a Java program using a stack or queue (CLO 1)

ULO #2: Apply algorithm analysis techniques to linear data structures (CLO 3)

ULO #3: Critique the performance characteristics of stacks and queues (CLO 5)

Introduction

This unit is an introduction to recursion and sorting algorithms that use recursion, such as merge sort and quicksort. Recursion is a technique where your program breaks a large problem into smaller pieces, and then solves each smaller piece using the same process. An example of this is merge sort, which breaks a list into two halves, sorts each half, then uses a merge process to create a single sorted list.

Unit Learning Outcomes

ULO #1: Implement a Java program using recursion (CLO 4)

ULO#2: Critique the efficiency of various sorting algorithms (CLO 3)

Introduction

Trees are an efficient way to store data that allow for faster searching than linear structures. But there are ways of storing data that allow for even faster lookup. If you think about arrays, it takes the same amount of time to access the first element as the last.
