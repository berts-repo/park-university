# Final Exam Study Guide

Exam Format

This proctored exam is closed book and closed internet access, except for exam itself. However, you may bring handwritten notes on either a 5x7” card or half sheet of 8.5x11” paper. You may use both sides. You may use a small whiteboard or single sheet of blank paper in a transparent sleeve and a dry erase marker for scratch paper.

You should show your notes to the camera, two seconds per side before you begin and erase and then show your scratch work to the camera when you finish.

There are 15-20 questions, worth 150 points, including multiple choice, short answer, and coding. Questions will be displayed one at a time, but you will be able to move back and forth and may answer them in any order.

Do not spend a lot of time on a 5-point question and then leave a 50-point coding question blank because you ran out of time.

You have 2 hours to complete the exam. You may only take the exam once.  

During the exam, you will be expected to type in short programs/methods directly in Canvas. You will not be allowed to use a compiler, nor will you be allowed to type in another editor and copy/paste into Canvas. The code you type does not have to compile and run for full credit. Minor syntax mistakes are irrelevant. When typing your answers, set the style to ‘Preformatted’ and use spaces to indent. 

Exam Content

The exam covers all material in Units 1-7. The core learning outcomes for the course are

Given a problem description, construct a Java program that uses algorithms and data structures from the Java Collections Framework.

Given a problem description, construct a Java program that implements an abstract data type using generics and linked objects.

Analyze the efficiency of algorithms using Big-O notation.

Implement recursive methods and distinguish appropriate from inappropriate uses of recursion.

Distinguish the characteristics of data structures such as linked lists, stacks, queues, priority queues, trees, heaps, dictionaries and hash tables.

As a good review, you should consider doing the exercises in the text again. This reviews the concepts and also lets you get used to working with the code again. You should look over all of your assignments and make sure you understand what you have written and understand the feedback given by the instructor. Ask any questions you have early in the week so they can get answered.
