Your instructor may request you to submit the assignment project files as a Zip file so that they can run the code and grade the code within the project directly.   

Submitting Your Homework as a Word Document

Your instructor may require you to submit a Word document with your code, instead of a zip file. 

In this case, for each unit you will create a Word document, with your responses, code, and screen shots of your web pages. This takes less than 5 minutes to complete. 

Take a screen shot of the web page including the address bar in the browser. 

Only take screen shots of the pages that you work on that week. 

Do not include the desktop, just the browser window!   

Copy and insert the screen shots into your homework Word document file. 

Several screen shots may be needed if the page has interactive content. (So for example, if the web page contains a form, show the form in one screen shot and in another screen shot show the form was processed.)

Copy the entire HTML source code from your entire web page source file to the Word document file. Select all the code, and then copy and paste the code from your editor directly into Word! You can format the code as 10 pt.  

If your page also uses an external CSS or JS file, you should also copy and paste the code from your editor directly into Word! You can format the code as 10 pt. 

Save the Word document file as unitX.docx where X represents the unit number.  

Upload the Word document file into the assignment page.

You may submit the document as a PDF format if you do not have Microsoft Word.

2. Testing and Debugging Programs  

Common Errors

You are a beginning web developer and will make mistakes when you begin writing your first web program. For example, common mistakes beginners make is to include content-related tags in the <head> section rather than in the <body> section. The body section is where the content of the page is placed. Please make sure that you are properly nesting your tags and content.  

You are expected to test your homework before submitting the assignment. You are expected to make sure your code works, and uses correct syntax.

Conform to HTML and CSS Standards

Just viewing your web page in a browser is not sufficient for testing your web program. Browsers may allow deprecated or obsolete tags and attributes. In this course, deprecated and obsolete elements, attributes and CSS style rules are not permitted. Students should test their web programs to make sure they work and validate that the code is following HTML 5 and CSS 3 standards.  

Absolute and Relative URLs

Some students use absolute URLs to images on web sites. Some students use a hard coded path to their web site resources. This method below hard codes the path and will cause errors depending on how the web site is viewed. Therefore avoid this method. 

<img src="/images/myimage.png" alt="My alternate text" />

This method uses a relative path. This relative path from the resource to the current page. Therefore this is the preferred method to use! Follow this same process for inserting external JavaScript and CSS files. 

<img src="../images/myimage.png" alt="My alternate text" />

3. Documentation and Proper Coding  

You should review how to insert comments and documentation as directed from the book and readings. Make sure to review the coding conventions in the appendices for HTML, CSS and JavaScript. This applies to all HTML, CSS and JavaScript programs for the entire semester! 

You are also required to follow the best practices and coding conventions as directed in the book and readings. Use proper nesting and indentation for all code. Make sure to include in each web page meta tags that identify your the page title, keywords, description, date the page was last modified and the page author (your name). Include comments in the all code files (name, date, purpose of code, functions, variables). Each major section in your page should be documented. (header, footer, main content, asides).

Commenting Your Code

Comments in HTML use the format below.

<!-- Comments go between the markers shown -->

Comments in embedded or external CSS style sheets use the format below for single or multi-line comments.

/* Comments go between the markers shown */

Comments in JavaScript programs use the format below.

// Single line comment
/* Multi-line comments */

Use proper descriptive naming conventions for the IDs and names of elements, names of classes used in all code. Don't use any reserved words like name or id for variables or other elements. Use descriptive names like txtFirstName or btnSubmit.

Additional requirements may be added during the course when you learn about specific topics. For example, after you learn about favicons in class you'll want to add that to your header using the link tag. After you learn about images you'll be expected to add the alternate text attribute to all images in order to support the accessibility standards.

4. Creating Web Pages, CSS and Scripts

Each unit you will have a combination of HTML, CSS and JavaScripts to create  web pages. You will also create, validate and process forms and build interactive programs on your web pages with the JavaScript. You are expected when you create each web page to insert the basic web page elements as directed in the readings. You will need to do this for every web page you create in this course.  (Doctype, html, head, meta, title body).

There are multiple ways to program using HTML and CSS. Pay attention to the directions. If the directions says to insert a paragraph element or paragraph tag, they mean the same thing. The tag <p> is the physical representation of the element in the markup code. If the directions say to insert the paragraph tag or element, make sure to include both the opening and closing tags. <p></p>.

Some programmers use <p> tags with line breaks <br> and others may use multiple <p> tags. There is a slight difference in the line height between the content areas, but they are both acceptable HTML elements. So you could use either one unless the directions specifically indicate a specific tag or element needs to be used.

There can be many solutions to the homework. There is a right and wrong answer. For example, your assignment may ask for a minimum of 3 heading elements. Having less than 3 would not fulfill the assignment requirements and could result in a loss of points. Similarly, a <li> tag could be used to display content as a list or as a menu. If you are asked to create a list, you should use the list item to display a list. However, a <li> tag is also sometimes used to create buttons or links in a menu. In that case, the directions would indicate to create a menu and not a list. So make sure to review the directions carefully!
