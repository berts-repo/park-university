What to Expect Each Week in the Homework

For the first four weeks you will read 2 chapters each week and will complete at least 2 web pages along with additional CSS style sheets. When we get to JavaScript, there is only one chapter each week and generally you have just 1 page (and sometimes the index page needs some work) along with CSS and JS files. 

For each web page, you need to provide the basic skeleton code. This is often the same in all of your pages. For example, navigation on web sites is usually the same because you want to have consistency for your visitors. Likewise, the CSS style rules are the same, because you want a common color palette, design and layout on your site. This is true for your homework too. 

For every homework throughout the entire course for every web page you create:

Let's look at the first at the skeleton code for any web page.  

The HTML Core Elements are sometimes called the skeleton code because all pages in your site will have the same elements. 

In the editor, insert the core elements as directed in the readings including the doctype, head,  meta, title and body elements. 

Doctype is always for HTML version 5. 

Always give your web site a title, by filling in your web site name inside the title tags.

Meta tags:

Replace the content [  ] with your content! 

Always change the author to your full name (Your first and last name). 

Always set the viewport to width=device-width, initial-scale=1.0 as shown below.

Always change the keywords, description and editor to something fitting for your web page.     

Make sure to CHANGE the description, title and keywords to something descriptive!

Example Skeleton Code:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="utf-8">
    <meta name="description" content="Put something about your site here">
    <meta name="keywords" content="Comma, separated, list of key words, that describe, your site">
    <meta name="author" content="John Smith"> 
    <meta name="generator" content="Brackets">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Vincent Van Gogh Gallery</title>
</head>
<body>
</body>
</html>

When you are finished with the skeleton page, you can preview the page in the browser. Nothing shows up because there is nothing in the body section of the page!  You will add that each week. 

Documenting Your Code

You are expected to comment in your code. The Tutorials in Chapter 1, at the end of the chapter, have information about documenting code files. Comments should be used to help organize your code and program logic. Comments are always to be included in the head and body sections of the page. 

You must always cite your image sources or any sources used, in the comment in the html code. Comments in HTML, CSS and JS use different ways to insert the comments. All files must be commented. All parts of the program need to be documented, including the external and embedded code. Documenting your name is not enough to receive points. You must document the code. Maintaining code is costly, labor intensive and can be a problem, as developers who write the program may not be the ones who maintain the program.
