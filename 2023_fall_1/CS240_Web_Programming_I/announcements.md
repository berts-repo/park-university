## Final Exam is available
*Posted: 2023-10-03T01:51:54Z*

Class 

It's been a busy term! You have come to the last week of class. The time really flies when you are taking online classes. 

Just a few additional reminders. The exam is in Honorlock, so you can only access it from the main Honorlock menu. 

Second, you must follow the instructions on the exam. Failure to follow instructions can be a reason to fail the exam. Cheating is not tolerated. So, I do allow you one handwritten notes page (2-sided is fine) and you can write as small as you want. But no typed pages and you need to write them! The point is to learn, and not to have to rely on the notes. It's not a test of your notes, but on what you learned in the class. 

Tips: Know the difference in writing comments in CSS, HTML and JS!

The first part are multiple choice with about 3 short answer. Don't take more than 15 minutes on this section. 

There are 4 problems where you need to write code. 

1. You have an HTML debug question. Given some HTML code, debug the code and fix the errors which may include using the style attribute. Do not take long to complete this! Review week 1 and 2. 

2. You have to create a CSS style sheet. Syntax counts! Know your properties. Make sure to use the CSS names, not JS!  Which properties? Make a list of the properties covered in the book. That's the best place to start! W3schools also has a list as well. Make sure you can create a rule that uses html selectors, class selectors and id selectors and pseudoclass. Know how to insert a comment.

3. You will create a short form. You don't need the whole page code. Focus on the form and form fields and properties. Know how to include html validation properties such as: required, min, max, size, max-length, pattern, etc. You can find this in the book too as well as w3schools if you don't know what html validation is. It is NOT javascript validation!!! Know how to insert a comment. Make sure you know the basic form tag (with id, name, action and method attributes), input tag (with type and id attributes, size, maxlength, readonly, required, style) as well as the select tag (dropdown list) and buttons. Know how to use the onclick event handler too. 

Validation:  [link: https://developer.mozilla.org/en-US/docs/Web/HTML/Constraint_validation] https://developer.mozilla.org/en-US/docs/Web/HTML/Constraint_validation

Attributes:  [link: https://www.w3schools.com/html/html_form_attributes.asp] https://www.w3schools.com/html/html_form_attributes.asp

HTML Form elements:  [link: https://www.w3schools.com/html/html_form_elements.asp] https://www.w3schools.com/html/html_form_elements.asp 

4. Create a short JS program. ONLY use JS in this activity. No HTML, No CSS! Know how to insert a comment. 

Retrieve the values from the form. Most of you are using getElementById, which is the easiest way for beginners. 

Do some calculation with the form. Make sure you know how to convert data types. 

Know how to put the results in the alert() window and in the page. (Look up innerHTML). Don't forget in the alert() you cannot use <br> to insert blank lines you need to use the character encoding "\n".  

Every term I change cases and examples, properties and directions. However you will see that these are the same things we test on every term. So focus on what I provide to you! 

Don't forget the homework should help you prepare too!

If you have any questions, you can contact me in my inbox.

How to Debug Your Code

Some students wrote the html, then tried to do all the processing in one sitting. That doesn't work. You should always do testing, slowly, one field at a time. Once your structure worked then go back and process the second field and so on. That was the mistake most students made. The other common mistake is to use third party code or to use the console log. The log can be useful, but, when you are learning, alert("") works just as well. I like to use alert() because it's integrated in the code. Once I make sure the code worked, I just comment out the alerts. You can't do that with console.log. Again, alert() is useful for debugging and learning.

Chapter 1 Review question 6

What is the name of the HTML element that generates the smallest heading?

The name of the HTML element that generates the smallest heading is <h6>. According to Mozilla (2023), these HTML tags go from h1 to h6. H1 being the largest and boldest. Here is an example of what they could look like in a web page. The web site has a nice tool to allow you to test out the examples. As you can see, it's easy to see the hierarchy of the headings and the different font sizes. 

I also learned from Mozilla (2023) that the headings are not just about the size and boldness, but also have meaning. Headings can be used to quickly locate the major sections of the web page. WebAim calls this the 'semantic structure' of the web page.  Headings are used to indicate major sections and subsections in the web page.  The h1 is typically used as the major heading for the page. My example below can help illustrate this point.

Headings are also important to visitors that have visual deficits. According to WebAim, people with visual disabilities can use special software called a screen reader to read aloud the page content. The headings help the user to navigate through the page. To comply with accessibility requirements, headings should be used in web pages.
