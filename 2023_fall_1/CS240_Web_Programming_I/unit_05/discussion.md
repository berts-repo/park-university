Review Question #9

What escape sequence should you use to display a double quote within an alert dialog message?

When we call the alert method, we can put a string into the message box. The issue is when we need to add the “ character, this is because the method requires this syntax: alert(“some message”), so if we need to add a message that is quoted, it will assume that the first quotes are in sequential order: alert(“the fallowing word needs quoted: “test””). To fix this we can use the \ character: alert(“the fallowing word needs quoted: \“test\””), this way the compiler knows which quotes to include in the message.

According to W3School there are 5 more escape characters besides the functional examples provided in the book (\’ and \” and \\ and \n). The 5 escape functions besides the ones previously listed are not supported in all browsers. The original purpose was for typewriters and doesn’t make much sense for HTML. The 5 less used functions are: \b (Backspace), \f (Form Feed), \r (Carriage Return), \t (Horizontal Tabulator), and \v (Vertical Tabulator).
