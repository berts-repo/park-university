# Final Exam Review

Learning Outcomes covered in the Final Exam: 

The final exam covers all four outcomes. 

Demonstrate creativity and problem-solving skills.

Analyze web programs in order to test, debug, and improve them.

Evaluate web pages and web programs to ensure that they use proper coding conventions and documentation.

Create web pages and web programs that use: HTML5, cascading style sheets, dynamic HTML, JavaScript, forms with controls, and Canvas.

Details

Please note the final exam is required to be proctored through the university approved proctor system. Please follow the directions in the syllabus.

This is a 2-hour final that can only be taken once.

The final exam is CLOSED BOOK.

The final exam is open notes. 

You may have 1 piece of paper with handwritten notes (both sides). 

You are not permitted to write on this paper during the exam!

No scratch paper is allowed.

Questions will be presented one at a time. You may still skip questions and return to them later.  

Additional Rules:

You will not be permitted to view the source code of any web pages or use the Internet during the exam. 

You will not be able to use an editor during the exam. The programs you write during the exam must be typed directly into the exam in Canvas. 

The exam is limited to 2 hours. You will want to work fast as you don't have time to look up answers! The first 15 questions are objective and you should spend no more than 10-15 minutes on that part. The second part has short essay questions about the concepts and syntax you learned. The last part you are going to be required to create short programs, using HTML, CSS and JavaScript. You should know how to write and insert JavaScripts (embedded and external), write and insert CSS style rules, (inline, embedded, external), and how to insert and use HTML elements, attributes and CSS style rules. Make sure to understand what HTML and CSS mean. 

The final exam is cumulative and therefore covers material from all units! If you are not prepared, it will be more difficult for you. Reviewing your homework is a great place to start. But also review your textbook and the lessons. So keep that in mind.

I advise you 'not' to take it until you do a review. Make sure you 'read' the directions carefully. You will get 1 question per page. The final exam is worth a large part of your grade. (Refer to the syllabus). So keep that in mind. 

Format

Part 1 are multiple choice and true/false questions. They cover basic things you should know based on the activities and readings as well as the homework activities. (Questions 1  - 15)

Part 2 are short answer and problem solving and worth more points . (Questions 16  - 18)

Part 3 are sample web programs that you are expected to write (Questions 19  - 22)

Content

Things you should be able to do: 

Write HTML code using HTML 5 using correct syntax. Make sure in all of your code, you use proper nesting of code. That means you may have to use the 'indent' tool on the toolbar to indent parts of your code where it's appropriate. 

Make sure you know what HTML 5 is, about browser wars, who created html and http, all of the tags in the page, including deprecated elements. Example elements include meta, title, headings, paragraphs, images, div tags and other elements as well as HTML 5 elements including header, footer, asides, articles and sections, nav and menu. You should know how to format color using hexadecimal and RGBA values.You need to know what are html 5 new elements and what tags are deprecated. You need to remember your syntax, including inserting all of the form field elements we covered. You need to remember how to set the id attributes and pick descriptive names as discussed in the readings and lessons. Quite a few questions will be on basic html and basic web concepts.

Write CSS rules using correct syntax. Be able to describe the difference between inline, embedded and linked/external CSS style sheets. You should be able to implement an external CSS file in a web page using the link element. You need to know the difference between syntax for style rules inline, embedded and external. You need to know how to refer to elements by their selectors, class and ID in the style rule, and the pseudo-classes that were covered in the readings such as :hover and :before. 

You should be familiar with all of the basic style rules and values such as font family, style, size, color, decoration, as well as borders, alignment, background properties, shadow and other properties. Creating the Form

You should be able to create a form from scratch. That means everything from the form tag, to the submit buttons and calling the function. Don't forget to know about things like the legend and labels for the controls and the attributes.

Validating the Form

Make sure to review how to 'validate' the form using HTML as well as how to format it with the new pattern attribute and regular expression. We covered HTML validation in the lessons, but not in the homework. You also need to know how we validated form fields with JavaScript. You need to know how to insert scripts into a web page.

You don't need to 'write out' the regular expressions by hand but you should have encountered several of them in your homework such as phone numbers, email and web addresses, zip code, country codes etc. Look up how to use these tools on W3Schools.com as your starting place. They will be common fields listed above!

You will have to be able to validate the form. I won’t tell you if it’s html validation or javascript validation until you are reading the question, so study both.

Processing the Form

You’ll need to remember how to retrieve values not only from text boxes and textarea elements but also dropdown lists and checkbox/radio buttons.

You will also need to be able to process the form by doing something with the form fields, like showing it the web page in a div tag or other element or in a window dialog box. Don't forget if you display it in the alert box you have to remember the alert window may need something different than the html tags.
