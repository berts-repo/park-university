## Week 4
*Posted: 2022-01-31T06:00:09Z*

Dear Students,

This week, we will explore the topic of malware. We will be able to categorize different malware types and understand the payload and lifecycle of different malwares. All the tasks are listed in  [link: https://canvas.park.edu/courses/65346/pages/unit-4-overview] Unit 4: Overview. [link: https://canvas.park.edu/courses/64007/pages/unit-4-overview] 

In addition, we will taste the flavor of attack by performing a hands-on lab. Specifically, we will perform a privilege escalation attack. In privilege escalation, the attacker usually connects to the victim’s machine with his low privilege, but legitimate account. Then the attacker explores the victim’s machine and finds out vulnerabilities. By exploiting the vulnerabilities, the attacker can obtain a higher privileged permission, which usually is a root account, which allows the attacker to do anything on victim’s machine. There are a lot of videos give brief introduction to privilege escalation. Such as this one:

 [link: https://www.youtube.com/watch?v=7PpYavvu-6k] What Is Privilege Escalation?

However, performing a privilege escalation on a real system is easier said than done. The most difficulty part is to find out system vulnerabilities, which varies from system to system.  

In our lab environment, we have two machines, the Client and the Server. You will be asked to work as a hacker, who first connects to the Server’s machine from Client with the account “tom”.  Due to misconfiguration on the Server, you, the hacker, can modify some critical files and install a payload on the Server. As a result, when the victim (the administrator on the Server) triggers the payload, the attacker can obtain a root shell of the Server from the Client machine.

This lab requires some Linux file operation and file permission knowledge, which has been covered in section 3-9 of our Linux Unhatched course. If you are not comfortable using Linux, feel free to check back.

Since the lab requires you to connect to both Server and Client, read the instruction carefully to make sure each step should be done on the Server or Client. 

This week, you can still use the alternative lab environment: Practice Cloud Store. (store.edaas.net). 

If you are using Practice Cloud Store, make sure to select the lab "Privilege Escalation (Free)" and use the lab instruction provided in the Practice Cloud environment. The instruction can be found by clicking the "View manual" button when entering the lab environment. 

If you are using vPark environment, use the lab instruction on the page  [link: https://canvas.park.edu/courses/65346/assignments/1929486] Unit 4: Lab - Privilege Escalation.

No mater which environment you are using, make sure that you did not miss points listed in the Rubrics.  If you are using Practice Cloud, follow the task number like "Task X.X on P.C."; if you are using vPark, follow the task number like "Task X in vPark".
