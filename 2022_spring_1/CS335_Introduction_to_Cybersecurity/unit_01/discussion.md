Hello class,

I chose to look into the recent vulnerability attack of the Log4Shell and Log4j, and can be tracked as CVE-2021-44228. (ThreatWire) The attack has a severity threat of 10 out of 10, and is considered a huge problem as it is used across many applications. Log4j is an open source Apache utility used by Java, and is built into many different apps. Log4j logs user inputs and preforms network lookups within the Java Naming and Directory Interface. The exploit is bad because it catches messages in a URL, and it can execute code with full privileges. To solve the problem, all the apps have to be updated and anything that uses Java, so companies have to go through about everything.
