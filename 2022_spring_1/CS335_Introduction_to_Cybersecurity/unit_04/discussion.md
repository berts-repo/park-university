The vulnerability I want to focus on is software with an end-of-life date. What makes this a risk is the developers are no longer developing patches for backdoors and bugs. This leaves the systems at risk for any unknown vulnerabilities. Running end-of-life software is one of the most common vulnerabilities in an enterprise. (Sethunathan, 2021)

According to BugTraq article in 2020, Microsoft Server 2003 R2 shipped out with .NET framework 2. (Kanthak) The .NET Framework 2.0 is out of date because Microsoft retired the Secure Hash Algorithm, SHA-1, on 08-03-2020. (Atikmarpari) They also shipped it out with Visual C 2005, which reached its end-of-life on 04-12-2016, and the latest update for the Visual C++ runtime was published 06-04-2011. (Kanthak)

You can combat this my always checking to see if your software is up-to-date.
